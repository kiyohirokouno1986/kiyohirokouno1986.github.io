# 社長秘書アプリ 設計メモ（v0.1）

社長の秘書のように「先回り」してスケジュールを管理し、Slack と連携して活用するためのツールの設計ドキュメント。

## コンセプト

「スケジュール**先行**管理」= 単なるカレンダー表示ではなく、秘書が先回りして気を利かせること。

- 📅 **先読みブリーフィング** … 来週/今日の予定と要注意点を自動でまとめて通知
- ⏰ **リマインド** … 次の予定・準備物を事前に通知
- ✍️ **対話で予定操作** … 音声メモやチャットからカレンダーへ登録
- ⚠️ **無理検知**（将来） … ダブルブッキング・移動時間不足・連続会議を警告

## 決定事項

| 項目 | 決定 |
|---|---|
| 予定の本体 | **Google カレンダー** |
| 目玉機能① | **来週の予定の先読みリマインド通知** |
| 目玉機能② | **PLAUD（プラウド）AI の音声メモからタスクを抽出 → カレンダー登録** |
| ホスティング | おまかせ → **Google Apps Script (GAS) 中心**を推奨採用 |
| PLAUD 連携方式 | **Claude in Chrome でブラウザから要約を読む**（API は作らない） |
| 登録前の確認 | **必ず社長の確認を取ってから登録** |

## アーキテクチャ

2つの役割を分担する。

```
【常時稼働パート】GAS 秘書エンジン
  └ 毎週トリガー → 来週の予定を先読み分析 → Slack通知（機能①）
  └ doPost受信 → カレンダー登録 → Slack確認（機能②の登録実行係）

【オンデマンドパート】Claude in Chrome
  └ 社長/秘書が「PLAUDメモ確認して」と指示
  └ ブラウザでPLAUDを開いて要約を読む
  └ タスク・日時・場所を抽出
  └ Slackで「これで登録しますか？」と提案 ─┐
                                          │ OK
  └ 確認OK → カレンダー登録 ←─────────────┘（機能②）
```

設計意図：「読む（Claude in Chrome）」と「定刻に動く・確実に登録する（GAS）」を分離する。
ブラウザ常駐は不安定なため、登録の確定処理は GAS 側に持たせて取りこぼしを防ぐ。

### なぜ GAS か

| 要件 | GAS での実現方法 |
|---|---|
| 常時稼働の定期実行 | 時間主導型トリガー（毎週/毎朝） |
| Google カレンダー読み書き | `CalendarApp` で直接アクセス |
| Slack 通知 | Slack Incoming Webhook / Bot に HTTP POST |
| 外部からの受信 | `doPost`（WebアプリURL） |
| AI でタスク抽出 | Claude API（Anthropic）を HTTP で呼ぶ |
| コスト | 無料枠で十分 |

将来「複数役員で使う」「Web 画面が欲しい」となったら、GAS を部品として残しつつ本格 Web アプリへ移行できる。

## 機能①：来週の予定 先読みリマインド（MVP 第一弾）

- **トリガー**：毎週金曜 17:00 ＋ 日曜 20:00（週末に翌週を把握できるタイミング）
- **処理**：翌週（月〜日）のイベントを取得 → Claude で秘書口調に要約
- **気利きポイント**：
  - 会議の多い日／予定ゼロの日を明示（「水曜は終日空き → 集中作業向き」）
  - 朝イチ・夜遅い予定を注意喚起
  - タイトルの `[要準備]` 等のタグを拾って準備リマインド（任意）
- **Slack 出力イメージ**：
  > 📋 来週のご予定です（6/22〜6/28）
  > ・月 10:00 役員会議 ／ 15:00 田中社長と会食
  > ・火 終日空き → 集中時間に最適です
  > ・木 9:00 と 21:00 に予定。長い一日になります、ご留意を

## 機能②：PLAUD 音声メモ → タスク → カレンダー登録

1. PLAUD で録音 → アプリ内に要約ができる
2. 社長/秘書が Chrome で PLAUD を開き、Claude に「今日のメモからタスク出して」
3. Claude が要約を読み、**予定候補リスト**を生成（例：`6/20 14:00 山田さんと打合せ @本社`）
4. Slack に候補を提示 → **OK/「登録して」で確定**
5. 確定分だけ Google カレンダーに登録 → Slack で「3 件登録しました」と報告

- 曖昧な日時（「来週あたり」等）は登録せず確認質問に回す。
- 誤登録ゼロを最優先。

## ロードマップ

1. **フェーズ 1（MVP）**：機能①の来週リマインドを GAS + Slack Webhook で実装
   - GAS プロジェクト作成、Calendar 読み取り、Slack 通知、週次トリガー設定
2. **フェーズ 2**：機能②の登録実行係（GAS `doPost` でカレンダー登録 + Slack 確認）
3. **フェーズ 3**：Claude in Chrome での PLAUD 読み取り運用フローを整備
4. **フェーズ 4（将来）**：無理検知・朝のブリーフィング・振り返りレポート

## 必要な準備物（実装着手時）

- Google アカウント（社長のカレンダーにアクセスできるもの）
- Slack ワークスペースの Incoming Webhook URL もしくは Bot Token
- Claude API キー（タスク抽出/要約に使う場合）
- PLAUD のログイン（Claude in Chrome で開くため）
