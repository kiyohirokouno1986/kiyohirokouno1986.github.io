# Auka × LINE AIエージェント 設計ドキュメント

家を建てたい夫婦のLINEグループにAuka公式LINEを招待するだけで、家作りの相談・サポート・日程調整までをAIが伴走するサービスの設計集。

## 背景

- 家を建てたい夫婦は、検討中にLINEグループ（または2人トーク）で情報共有していることが多い
- そこにAuka相談カウンターの公式LINEを招待するだけで、AIが第三のメンバーとして相談相手・調整役・知識補完者になる
- 2026年4月20日にLINEヤフーが「Agent i」を発表、同年夏に「LINE OA AIモード」を提供予定
- ただしMVPはMessaging API + Claude Agent SDKで先行構築し、夏以降にLINE側のメモリ機能・パーソナライズ機能を取り込む方針

## 設計の前提（調査済み事実）

- LINE公式アカウントは **招待されないとグループに入れない**（自発参加・他者招待ともに不可）
- **1グループに公式アカウントは1つまで** → 先取り独占ポジションが取れる
- グループ内のファイルは1週間、画像/動画は2週間で消える → **Webhook受信時に即時取得・自社保存が必須**
- LINE OA AIモードは2026年夏予定 / Agent i Bizは2026年8月予定
- メモリ機能は2026年6月までにLINE側で実装予定

詳細は調査ログ（このリポジトリでは省略、別途共有）を参照。

## 設計の核となる判断

1. **「招待されないと入れない」を弱点でなくコンセプトの核に据える**
   - 夫婦が能動的に招き入れた相談相手という心理的ポジションは強力
   - 営業導線では一切使わない／呼ばれた時だけ話す沈黙力を初手で約束

2. **「1グループ1アカウント」は競合排除の盾**
   - 先に席を取れば、夫婦の検討期間中は他工務店・HM・他相談サービスを物理的に排除できる
   - 営業設計の核に格上げ

3. **ハイブリッド実装パス**
   - 今：Messaging API + Claude Agent SDK で先行構築
   - 夏以降：LINE OA AIモードのメモリ・パーソナライズを取り込む
   - オーケストレーション層を抽象化しておき、後で置換可能に
