# オンボーディングUX設計（C-3）

> Auka × LINE AIエージェント / 家作り夫婦グループ常駐AI
> v0.1 (2026-04-25)

## 0. 設計の前提（調査で確定済み）

- **公式アカウントは招待されないとグループに入れない**（自発参加・他者招待ともに不可）
- **1グループにつき公式アカウントは1つまで**（→ 先取り独占ポジション）
- グループ内のファイルは **1週間**、画像/動画は **2週間** で消える
- LINE OA AIモードは2026年夏予定。**今はMessaging API + 自社AIで先行実装する**前提

→ オンボーディングのKPI：「夫婦が **自分たちのグループにAukaを招き入れる** までの離脱率を最小化する」

## 1. ペルソナと心理状態

### Primary: 30代前半夫婦・初めての家作り
- 既に夫婦の検討用LINEグループ（または2人トーク）を持っている
- 工務店・HM・SUUMO・SNS情報で混乱している
- 夫婦間で予算・優先順位がズレ始めている自覚がある
- 「客観的に整理してくれる第三者」を欲しているが、**営業はされたくない**

### 心理的な摩擦
| 摩擦 | 設計対応 |
|---|---|
| 「公式アカウントを夫婦グループに入れるのは抵抗がある」 | 「呼ばれた時だけ話す」を初手で明示 |
| 「営業電話みたいに来そう」 | 課金構造を冒頭で説明（無料・営業しない・選ぶのはあなた達） |
| 「データを抜かれる気がする」 | プライバシーポリシーを1画面で要約し、退出方法も明示 |
| 「使い方がわからない」 | 招待〜初回会話までを **動画＋GIF** でガイド |

## 2. ファネル全体像

```
[認知]
  ↓  (Webサイト / 工務店経由 / SNS / QR)
[1. 友だち追加]
  ↓  ワンタップ
[2. 1:1ウェルカム]               ← ここで Auka の人格と利用ルールを刷り込む
  ↓  (任意プロファイリング 0〜2問)
[3. グループ招待ガイド]          ← LIFFで摩擦最小化
  ↓
[4. グループ参加イベント受信]
  ↓  (Bot自己紹介 30秒で読める長さ)
[5. 沈黙モードに入る]            ← 押し付けない設計
  ↓  (夫婦の最初のメンション/質問を待つ)
[6. 初回応答]                   ← ここで信頼を勝ち取る
  ↓
[7. 継続的伴走]
```

各段で離脱率を計測。**特に[3]→[4] が最大の崖**になる。

## 3. 各ステップの詳細設計

### Step 1: 友だち追加

- エントリポイントは複数：
  - Aukaサイト内QR / URL
  - 工務店・住宅展示場のチラシQR（提携先からの導線）
  - SNS広告
  - 既存Auka顧客からの紹介URL（紹介者特典あり）
- **ワンタップで追加完了**（LINEの標準導線に従う）
- 追加直後にすぐに次のステップへ

### Step 2: 1:1ウェルカム（10秒以内に最初の発話）

最初のメッセージは **3つのバブル** に分割：

```
[バブル1: 自己紹介]
こんにちは、Auka相談カウンターのAIアシスタントです。
家作りのことを、夫婦お二人と一緒に考えるお手伝いをしています。

[バブル2: 利用方針 ←これが最重要]
私は、お二人のLINEグループに招待していただいた時だけ、
そこで会話に参加します。呼ばれていない時は黙っています。
営業の電話やしつこい連絡は一切しません。

[バブル3: 次のアクション]
よろしければ、お二人のグループに私を招待していただけますか？
↓ 招待方法をご案内します
[招待方法を見る] (Quick Reply)
[まずは1人で相談したい] (Quick Reply)
[使い方を詳しく聞く]   (Quick Reply)
```

設計判断：
- **1問目は強要しない**。プロファイリングは「グループ招待後」に回す
- 「営業しない」を最初の3バブルで言い切ることが信頼の前提
- Quick Replyで3択にすることで「どこから始めても良い」を体現

### Step 2.5: 軽量プロファイリング（任意・スキップ可）

「使い方を詳しく聞く」を選んだ場合のみ、雑談の流れで2問だけ聞く：
1. 家を建てるの、いつ頃をイメージしていますか？（時期）
2. 主にどのエリアで考えていますか？（場所）

→ このデータは groupId と紐づける前なので **userId 単位で一時保存**、後でグループ参加時にマージ。

### Step 3: グループ招待ガイド（最重要・LIFF活用）

LIFF（LINE Front-end Framework）で専用ページを開き、**3画面のチュートリアル**：

```
[画面1] なぜグループに招待するの？
  → 「夫婦お二人の会話の文脈を理解した上でアドバイスできるから」
  → 一人ずつ別々に相談するより、二人が同時に同じ情報を見られる安心感

[画面2] どうやって招待するの？
  → 30秒のスクリーン録画GIF（実機操作）
  → ステップ：
     1. 夫婦のLINEグループを開く
     2. メニュー → 招待 → Aukaを選択
  → ※「公式アカウントは1グループ1つまで」を注釈で明示

[画面3] 招待ボタン
  → タップすると LINEの「友だちを招待」シートが開く
  → 夫婦のグループを選択して招待で完了
  → 招待完了後、自動でAukaトークに戻る
```

設計判断：
- 文章ではなく **GIF/動画** で動作を見せる（読まずに真似できる）
- LIFFから直接グループを選択できるなら理想だが、LINE仕様上は標準シート経由になる可能性大
- 「グループまだ作ってない」夫婦向けに、「先に夫婦のグループを作る」ガイドも用意
- このステップでの離脱者向けに、**24時間後に1回だけ**「グループ招待してみませんか？」とプッシュ

### Step 4: グループ参加イベント受信（Webhook: join）

`join` イベントを受信したら、即座に：

1. groupId を発番・DB保存
2. メンバーリスト取得API呼び出し（夫婦2人＋親など同居家族の可能性）
3. 1:1で取得済みのプロファイル（招待者ベース）をグループに紐付け

### Step 5: グループでの自己紹介（30秒で読める長さ）

```
[自己紹介バブル]
こんにちは、Aukaです。お招きいただきありがとうございます。

私はお二人の家作りの相談相手として、こちらに常駐します。
ただし、基本的には黙っています。

🗣 話しかけてほしい時：
   「@Auka」でメンションするか、
   「Auka」と呼びかけてください。

🔇 一旦静かにしてほしい時：
   「Auka 黙って」と書いてください。
   再開は「Auka 戻って」で。

🚪 退出したい時：
   「Auka 退出」と書いてください。
   いつでも再招待できます。

何かあればいつでもどうぞ。
```

設計判断：
- 自己紹介の **直後に黙る**（夫婦の会話を邪魔しない）
- 操作方法を3つだけに絞る（呼ぶ・黙らせる・退出）
- 絵文字は最小限、敬語、押し付けない人格

### Step 6: 初回応答（最重要の信頼獲得ポイント）

- 夫婦の最初のメンション/質問に対し、**5秒以内**に返答（タイピングインジケーター活用）
- 最初の3往復は特に丁寧に：
  - 質問の意図を一度復唱して確認
  - 回答は **結論 → 根拠 → 注意点** の順で簡潔に
  - 「もし違ったら教えてください」を必ず添える
- 売り込みは絶対にしない。来店誘導CTAは初回応答では出さない

### Step 7: 継続伴走

- 検討フェーズ（初期検討→土地→設計→契約→着工）を自動推定
- フェーズ移行時に控えめな提案（「次は〇〇を考える時期かもしれません」）
- 月1回程度、論点整理レポートを能動配信（夫婦が望む場合のみ・要オプトイン）

## 4. プライバシーと信頼設計

### 必須開示（1:1ウェルカムから1タップで読める場所に）
- 何のデータを取得するか（メッセージ・画像・ファイル・groupId・userId）
- 何に使うか（応答生成・カウンセラーへの引継ぎ・サービス改善）
- 第三者提供の有無（しない、ただし夫婦の同意ある工務店紹介時のみ）
- 保存期間（〇〇年、退出時は即時削除可能）
- 退出方法（「Auka 退出」「データも削除して」で完全削除）

### 「データも削除して」の処理
- groupId配下のメッセージ・ファイル・プロファイルを物理削除
- 監査ログのみ保持（誰が削除を依頼したか）

## 5. 計測すべきKPI

| ステップ | KPI | 初期目標 |
|---|---|---|
| 友だち追加 | 追加完了率（流入元別） | 流入元次第 |
| 1:1ウェルカム | ウェルカム到達後の離脱率 | < 30% |
| **招待ガイド到達** | LIFFを開いた率 | > 50% |
| **グループ招待完了** | LIFF開いた人のうち招待した率 | **> 40%** ←最重要 |
| 初回応答 | 招待後24時間以内の初メンション率 | > 60% |
| 継続 | 招待後7日後にもアクティブな率 | > 50% |
| 来店誘導 | 招待後30日以内の来店予約率 | > 15% |

「グループ招待完了率」が0.4を切ったらUX全体の見直しが必要。

## 6. エッジケースと対応

| ケース | 対応 |
|---|---|
| 既存グループに別の公式アカウントがいる | 招待時にエラー → ガイドで「他公式アカウントを抜く or 別グループ作成」を提示 |
| 一人だけが追加して招待しない | 1:1相談として継続。3日後にリマインド1回 |
| グループ参加後、1週間誰も話しかけない | プッシュ禁止（沈黙力）。来店予約への動線リッチメニューだけ用意 |
| 喧嘩中の夫婦の会話 | 介入禁止。メンションされた時のみ「論点を整理しましょうか？」とだけ提案 |
| 親や工務店も同じグループに入っている | メンバーリスト取得時に「夫婦」を識別（ニックネームor1:1で確認）。発言主体を区別して文脈管理 |
| 退出後の再招待 | 過去の文脈は **デフォルトで引き継がない**。「以前の続きから始めますか？」と確認 |

## 7. 次に詰めるべき項目

1. **LIFFでの招待UI**：実際にどこまで摩擦を減らせるかLINE仕様調査が必要（次回 LINE Developers管理画面で確認）
2. **自己紹介文のA/Bテスト案**：少なくとも3パターン用意して初期ベータで検証
3. **「夫婦」識別のロジック**：1:1での自己申告 / メンバー名のヒューリスティック / 会話内容からの推定 のどれを軸にするか
4. **オフライン引き継ぎ**：実店舗カウンセラーが「グループの履歴を要約で受け取る」UIをどこに置くか（管理画面 / Slack連携）
